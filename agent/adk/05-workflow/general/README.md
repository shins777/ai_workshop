# ADK 기본 워크플로우 - 에이전트 라우팅 (05-workflow/general)

이 예제는 사용자의 의도에 따라 적절한 하위 에이전트를 선택하여 실행하는 가장 기본적인 **Orchestrator-Workers** 패턴을 보여줍니다.

## 주요 구성 요소

### 1. 에이전트 정의 (`agent.py`)
- **`root_agent`**: 사용자의 질문을 분석하고 요약한 뒤, 지침(Instruction)에 따라 하위 에이전트를 호출합니다.
- **라우팅 로직**: 시스템 프롬프트(Instruction)를 통해 "긍정적인 비평은 `positive_critic`에게, 부정적인 비평은 `negative_critic`에게" 업무를 할당하도록 정의되어 있습니다.

### 2. 하위 에이전트 (`sub_agent.py`)
- **`positive_critic`**: 주제에 대한 긍정적인 리뷰를 담당합니다.
- **`negative_critic`**: 주제에 대한 부정적인 리뷰를 담당합니다.
- 각 에이전트는 출력값을 구분하기 위해 고유한 `output_key`를 가질 수 있습니다.

## 워크플로우 동작 방식
1. 사용자가 질문을 입력합니다.
2. 루트 에이전트가 질문의 의도를 요약합니다.
3. 질문 내용이 긍정/부정 비평을 요구하는 경우, 해당 하위 에이전트에게 업무를 전달합니다.
4. 하위 에이전트의 결과가 루트 에이전트의 최종 응답에 포함됩니다.

## 실행 방법
`05-workflow` 폴더에서 `adk web`을 실행한 후, 에이전트 목록에서 `general`을 선택하여 테스트할 수 있습니다.
