# ADK 에이전트 워크플로우 통합 가이드 (05-workflow)

이 디렉토리는 ADK(Agent Development Kit)를 활용하여 여러 에이전트를 조합하고 복잡한 업무 프로세스를 자동화하는 다양한 **워크플로우 디자인 패턴**을 다룹니다.

## 워크플로우 패턴 목록

### 1. [GENERAL (에이전트 라우팅)](./general/README.md)
- **패턴**: Orchestrator-Workers
- **설명**: 루트 에이전트가 사용자의 질문 의도를 파악하여 적절한 하위 에이전트에게 업무를 할당합니다. 단순한 업무 분담과 라우팅에 적합합니다.

### 2. [SEQUENCIAL (순차적 실행)](./sequencial/README.md)
- **패턴**: Pipeline
- **설명**: 에이전트들을 정해진 순서대로 실행합니다. 이전 에이전트의 출력 결과가 다음 에이전트의 입력으로 사용되는 단계별 프로세스에 적합합니다.

### 3. [PARALLEL (병렬 실행)](./parallel/README.md)
- **패턴**: Parallel Execution
- **설명**: 여러 에이전트를 동시에 실행하여 독립적인 정보를 빠르게 수집합니다. 병렬 작업 완료 후 결과를 취합하는 구조입니다.

### 4. [LOOP (반복 및 개선)](./loop/README.md)
- **패턴**: Iterative Refinement
- **설명**: 결과물이 만족스러울 때까지 '비평 -> 수정' 과정을 반복합니다. `LoopAgent`와 탈출 도구(`exit_loop`)를 사용한 품질 개선 루프를 보여줍니다.

### 5. [CUSTOM (사용자 정의)](./custom/README.md)
- **패턴**: Custom Logic
- **설명**: 표준 패턴 외에 개발자가 직접 에이전트 간의 상호작용과 상태 전이를 제어하는 세밀한 커스텀 워크플로우를 구현합니다.

---

## 실행 방법

모든 워크플로우 예제는 상위 폴더의 설정을 바탕으로 `adk web` 인터페이스에서 통합 테스트할 수 있습니다.

1. `05-workflow` 폴더에서 다음 명령어를 실행합니다:
   ```bash
   adk web
   ```
2. 실행된 웹 UI에서 로딩된 에이전트 목록 중 테스트하려는 패턴(`general`, `sequencial`, `parallel`, `loop`, `custom`)을 선택합니다.
3. 각 워크플로우가 정의된 순서와 로직에 따라 하위 에이전트들을 호출하는 과정을 실시간으로 확인하세요.
