# ADK 에이전트 도구 예제 (03-tools/agent_tool)

이 예제는 다른 에이전트를 도구(Tool)처럼 호출하여 복잡한 작업을 수행하는 **Agent as a Tool** 패턴을 보여줍니다.

## 주요 개념

- **AgentTool**: 다른 `Agent` 인스턴스를 ADK 도구 규격으로 감싸서 루트 에이전트가 호출할 수 있게 합니다. 이는 특정 분야에 전담된 하위 에이전트에게 업무를 위임할 때 매우 유용합니다.

## 주요 구성 요소

### 1. 하위 에이전트 정의 (`sub_agent.py`)
- **`positive_critic`**: 주제에 대해 긍정적인 비평을 작성하는 전문 에이전트입니다.
- **`negative_critic`**: 주제에 대해 부정적인 비평을 작성하는 전문 에이전트입니다.

### 2. 루트 에이전트 정의 (`agent.py`)
- **`AgentTool` 적용**: `tools = [AgentTool(agent=positive_critic), ...]` 와 같이 하위 에이전트를 도구 목록에 등록합니다.
- **라우팅 로직**: 사용자의 요청 내용에 따라 루트 에이전트가 어떤 비평가 도구(에이전트)를 사용할지 스스로 결정합니다.

## 워크플로우 동작 방식
1. 사용자가 "이 아이디어에 대해 긍정적인 면과 부정적인 면을 모두 알려줘"라고 요청합니다.
2. 루트 에이전트가 질문을 분석하고 `positive_critic` 에이전트(도구)와 `negative_critic` 에이전트(도구)를 각각 호출합니다.
3. 각 하위 에이전트는 자신의 지침(Instruction)에 따라 답변을 생성하여 루트 에이전트에게 전달합니다.
4. 일반적인 답변은 루트 에이전트가 처리합니다.
5. 루트 에이전트가 모든 비평 결과를 취합하여 사용자에게 최종 답변을 제공합니다.
